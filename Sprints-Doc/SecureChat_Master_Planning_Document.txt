Executive Summary
SecureChat is an ambitious open-source secure messaging platform designed to provide state-of-the-art end-to-end encryption with novel post-quantum cryptographic enhancements. This project serves dual purposes: as a master's capstone demonstrating original research contributions to secure messaging protocols, and as a foundation for a commercial-grade open-source application.
The platform implements a Hybrid Post-Quantum Signal Protocol, combining the proven security of the Signal Protocol (X3DH + Double Ratchet) with CRYSTALS-Kyber post-quantum key encapsulation. This approach ensures both current security and future resilience against quantum computing threats.
Project Objectives
• Implement Signal Protocol with hybrid post-quantum key exchange
• Deliver a functional web-based messaging prototype (Phase 1)
• Establish foundation for cross-platform expansion (Phases 2-3)
• Contribute original research to the field of secure messaging
• Create an open-source platform for community development

Project Overview
Vision Statement
To create a secure, private, and quantum-resistant messaging platform that empowers individuals and organizations to communicate without fear of surveillance or future cryptographic compromise.
Problem Statement
Current secure messaging applications face several critical challenges that SecureChat aims to address:
• Quantum Threat: Existing protocols using RSA/ECDH are vulnerable to future quantum attacks
• Complexity Barrier: Implementing secure messaging is prohibitively complex for most developers
• Platform Fragmentation: No unified solution exists across web, desktop, and mobile
• Trust Issues: Proprietary solutions require trusting vendors with cryptographic implementations
Proposed Solution
SecureChat addresses these challenges through a comprehensive technical approach:
Hybrid Post-Quantum Cryptography
The platform implements a dual-layer key exchange combining X25519 (classical) with CRYSTALS-Kyber-768 (post-quantum). This ensures security against both current and future threats while maintaining compatibility with existing infrastructure.
Signal Protocol Foundation
Building on the battle-tested Signal Protocol provides forward secrecy and future secrecy through the Double Ratchet algorithm, ensuring that compromise of long-term keys does not affect past or future message security.
Cross-Platform Architecture
Using .NET MAUI and Blazor WebAssembly enables code sharing across all platforms while maintaining native performance and user experience.


Technical Architecture
System Overview
SecureChat employs a client-server architecture where the server acts only as a message relay and key distribution center, never having access to plaintext messages or private keys.
Architecture Principles
• Zero-Knowledge Server: Server cannot decrypt messages or access private keys
• End-to-End Encryption: All messages encrypted on client before transmission
• Forward Secrecy: Compromise of current keys does not affect past messages
• Post-Compromise Security: System recovers security after key compromise
• Minimal Metadata: Server stores minimum necessary information

Cryptographic Stack
Key Exchange (X3DH with Post-Quantum Enhancement)
The Extended Triple Diffie-Hellman (X3DH) protocol is enhanced with CRYSTALS-Kyber to create a hybrid key agreement:
ComponentClassicalPost-QuantumPurposeIdentity KeyEd25519Dilithium-3Long-term identitySigned Pre-KeyX25519Kyber-768Medium-term key agreementOne-Time Pre-KeyX25519Kyber-768Single-use key agreementEphemeral KeyX25519Kyber-768Per-session key agreement
Message Encryption (Double Ratchet)
• Symmetric Ratchet: HKDF with SHA-256 for chain key derivation
• DH Ratchet: X25519 + Kyber-768 hybrid for ratchet steps
• Message Encryption: AES-256-GCM for authenticated encryption
• Header Encryption: AES-256-GCM for metadata protection
Additional Security Measures
• Key Derivation: HKDF-SHA256 for all key derivations
• Message Authentication: HMAC-SHA256 for integrity verification
• Padding: PKCS#7 padding with random length extension
• Timestamp Verification: Prevent replay attacks with bounded clock skew
